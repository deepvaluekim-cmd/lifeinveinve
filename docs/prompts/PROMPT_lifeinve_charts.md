# PROMPT — lifeinve_charts 데일리 페이지 생성기

> 목적: **하루치 섭스택 URL 묶음**(또는 **실적 이벤트 리스트**)을 입력받아,
> 라이프자산운용 브랜드를 입힌 **자기완결형 HTML 1파일**(정적 호스팅용)을 만든다.
> 원본 포맷은 must-charts.pages.dev 의 `daily_substack_*` 페이지(섭스택 데일리)를 계승하고,
> 여기에 라이프자산운용 컬러/폰트 브랜드킷을 덧입힌다.
> 부모 포맷 규칙은 [[must-charts-research-format]] 를 따른다.

---

## 0. 입력 (INPUT)

> **호스팅/구성 결정(2026-07-01 사용자 확정)**:
> - 호스팅 = **신규 프로젝트 "lifeinve-charts" 전용**. ⛔ must-charts 프로젝트에 **절대 얹지 않음**.
> - **섭스택 데일리와 실적 데일리는 완전 별도 페이지**로 각각 매일 산출(한 페이지 통합 금지).
> - 파일명 slug로 모드 구분: `...-substack-...` vs `...-earnings-...`.

이 프롬프트를 실행할 때 아래 중 하나를 받는다.

- **모드 A — 섭스택 데일리**: 그날 커버할 섭스택/뉴스레터 URL 목록 (5~50건), 채팅방 로그(선택). → 별도 페이지.
- **모드 B — 실적 요약(earnings)**: 그날 실적 발표 종목 리스트 (티커 + 발표 이벤트). → 별도 페이지.

공통 입력 파라미터:
- `date` (YYYY-MM-DD, 기본값 = 오늘)
- `sources[]` (커버한 매체/섭스택 이름 리스트 — 푸터·메타에 그대로 표기)
- `mode` = `substack` | `earnings`

## 1. 베이스 구조 (must-charts `daily_substack` 스켈레톤 계승)

4개 레퍼런스 페이지(2026-06-11/12/15/16)에서 공통으로 관찰된 **불변 스켈레톤**. 이 순서·블록을 그대로 유지한다.

### ① 헤더 블록
- 매체명: **`섭스택 데일리`** → lifeinve 판에서는 **`라이프 인베 차트 — 데일리`** 로 교체
- 타이틀: `라이프 인베 차트 — {YYYY-MM-DD}`
- 키커(상단 소제목): `LIFE ASSET · SUBSTACK 리뷰 {date}` (원본은 "MUST INVESTMENT · SUBSTACK 리뷰")
- 태그라인(고정 문구): `오늘 섭스택에선 무슨 얘기가 — 뉴스레터 한 면 요약`
  - earnings 모드: `오늘 실적에선 무슨 얘기가 — 실적 한 면 요약`
- 메타 라인: `{요일} · {YYYY년 M월 D일} | {소스 건수·매체 요약}`
  - 예) `목요일 · 2026년 6월 12일 | flow-substack-report 50건 · 채팅 포함`

### ② "오늘의 한 방" (Top takeaways)
- 3~4개 **번호형 헤드라인 불릿**. 각 불릿 = `티커 + 등락률(%) + 한 줄 핵심`.
- 그날의 가장 큰 무브/테제를 압축. (원본 공통 요소)

### ③ 본문 — 번호형 메인 아이템 (1~9)
각 아이템은 아래 소구조를 반드시 갖는다(원본 per-item 포맷):
1. **번호 + 티커** — 미국주 `$AAPL`, 한국주 `000660.KS` / `.KQ`
2. **긴급도 태그** — `긴급`(속보) / `화제`(트렌딩) 배지
3. **볼드 헤드라인** — 평서/의문형 (음슴체 프래그먼트 허용)
4. **서브 헤드라인** — 작은 폰트, 맥락 앵글 한 줄
5. **출처 라인** — `출처:` 접두, `·` 구분 2~4개 매체
6. **본문 2~4문단** — 요약·분석
7. **풀 인용(선택)** — 볼드 + 화자 귀속: `김민지: "MLCC는 아직 가격 인상까지 멀었어요"`

### ④ 정렬 규칙
- **시장 임팩트/긴급도 내림차순**: 글로벌·속보 → 한국 내수 → 기타.

### ⑤ 보조 섹션 (해당 시)
- `매크로` — CPI 등 매크로 코멘트 블록 (선택)
- `그 외 오늘` — 소소한 아이템 **속사포 불릿** 요약
- `오늘의 섭스택 채팅방 · 말말말` — 채팅방 담론, **채널별 서브섹션**, 볼드 화자명 + 인용, 스레드형 진행

### ⑥ 푸터 / 카비앗
- `라이프 인베 차트 · {소스 근거} · 자동 생성`
- 전체 소스 매체 리스트 나열
- **카비앗**: 자동 생성물 · 원문 요약이며 투자자문 아님 · 수치는 발행 시점 기준

### earnings 모드 매핑
- ②"오늘의 한 방" → 그날 실적 서프라이즈 TOP 3~4 (티커 + EPS/매출 서프라이즈% + 주가 반응)
- ③ 메인 아이템 = 종목별 실적 카드: `티커 · 분기 · EPS(예상 vs 실제) · 매출(예상 vs 실제) · 가이던스 · 컨콜 한 줄`
- 표(table) 사용 허용 (원본 섭스택 데일리는 무표였으나 실적은 표가 적합)

## 2. 브랜드킷 — 라이프자산운용 컬러 (Requirement 1)

> 출처: 공식 로고 SVG (`investlife.com/wp-content/uploads/2021/07/Frame-116~117.svg`) 에서 추출한 실제 fill 값.
> 공식 사이트(investlife.com)는 화이트 배경·미니멀 무채색 톤이며, **로고 마크는 그린 계열**.

| 역할 | HEX | 설명 |
|---|---|---|
| **Primary / Key (라이프 그린)** | `#608C34` | 로고 메인 그린. 헤더 바·H2 언더라인·티커 배지·강조선 |
| **Accent (라이트 그린)** | `#8FB85D` | 로고 하이라이트 그린. 호버·서브 강조·`화제` 배지 |
| **Ink (본문 텍스트)** | `#070203` | 로고 워드마크의 근흑색. 본문/제목 기본 텍스트 |
| **Background** | `#FFFFFF` | 공식 사이트 톤(화이트·여백 중심) |
| **Muted / 라인** | `#EDEFEA` ~ `#Df--E7E6E6` | 옅은 그레이-그린 보더·디바이더 (근사값) |
| **긴급 배지** | `#C0392B` (레드) | `긴급` 태그 강조용 보조색 (근사값, 브랜드 외 기능색) |

> ⚠️ 근사 주의: `#608C34`·`#8FB85D`·`#070203` 는 **공식 로고 SVG에서 추출한 정확한 값**.
> Muted 그레이/긴급 레드는 브랜드 가이드가 공개돼 있지 않아 **근사·기능색**이다.
> 공식 CI 컬러 문서 확보 시 교체할 것.

CSS 변수 예:
```css
:root{
  --life-green:#608C34;      /* primary/key */
  --life-green-light:#8FB85D;/* accent */
  --life-ink:#070203;        /* text */
  --life-bg:#FFFFFF;
  --life-line:#E7E6E6;       /* 근사 */
  --life-urgent:#C0392B;     /* 기능색 */
}
```

## 3. 폰트 / 타이포그래피 (Requirement 2)

> **정본 = 라이프자산운용 공식 웹사이트(investlife.com)에서 실측한 computed font-family.**
> (사내 pptx의 나눔스퀘어는 장표용 추정이었고, **웹 브랜드 서피스는 아래가 정확**.)

- **한글 본문/제목**: **`Noto Sans KR`** — investlife.com body·h2·h3·p 전부 이 서체(실측). Google Fonts, OFL 라이선스 → 정적 호스팅 CDN 임베드 가능(라이선스/CDN 문제 해소).
- **영문/숫자**: **`Open Sans`** (사이트 nav 실측; `Roboto`도 로드됨). Google Fonts.
- **웹폰트 로드(정적 호스팅용)**:
  ```html
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link href="https://fonts.googleapis.com/css2?family=Noto+Sans+KR:wght@400;500;700;900&family=Open+Sans:wght@400;600;700&display=swap" rel="stylesheet">
  ```
- **폰트 스택**:
  ```css
  font-family:"Noto Sans KR","Open Sans",-apple-system,"Segoe UI","Malgun Gothic",sans-serif;
  /* 숫자·티커 강조부는 Open Sans 우선: */
  font-family:"Open Sans","Noto Sans KR",sans-serif;
  ```
- 위계: 타이포 기반 위계(원본은 색 코딩 최소·타이포로 위계). lifeinve는 여기에 그린 언더라인/배지로 약한 색 위계를 추가.
- 티커·지표·%는 tabular 숫자(`font-variant-numeric:tabular-nums`) 권장.

## 4. 문체 규칙 (Requirement 3)

레퍼런스 4개 페이지 관찰 결과 = **혼합 레지스터**.
- **헤드라인·불릿·"오늘의 한 방"·"그 외 오늘"** → **음슴체 / 개조식 프래그먼트** (예: "코스피 -4.52% 폭락", "월요일 -15% 셀오프 온다").
- **본문 문단** → **한국어 완전문(평서문)** 서술체. (must-charts 본문은 줄글 한국어)
- 채팅방 인용 → 원발화 톤 보존, 볼드 화자명.

> 명확화: 사용자의 **balje(발제) 스킬은 전면 음슴체**이지만, **must-charts/lifeinve 본문은 한국어 줄글(완전문)** 을 쓴다.
> lifeinve는 **헤드라인=음슴체, 본문=완전문** 의 하이브리드를 따른다. 최대한 정돈·구조화(번호·배지·출처라인 고정).

## 5. 팩트 검증 (Requirement 4)

**모든 수치/사실은 웹 검증 + 출처 명시.** 무출처 수치 금지.

검증 체크리스트 (페이지 발행 전 전수 통과):
- [ ] 모든 등락률(%)·주가·시총·EPS·매출 수치 → 1차 소스(공시/IR/발행 섭스택) 재확인
- [ ] 티커 표기 정확 (`$US` / `NNNNNN.KS` / `.KQ`)
- [ ] 각 메인 아이템에 `출처:` 라인 존재 + 실제 커버한 매체명과 일치
- [ ] 인용문 화자·문구 원문 대조 (왜곡·창작 금지)
- [ ] 날짜/요일 정합 (`date`와 메타 라인 일치)
- [ ] 매크로 지표(CPI 등)는 발표기관 수치와 대조
- [ ] earnings 모드: 예상치(컨센서스) 출처 명기 + 실제치 공시 대조
- [ ] 추정/전망은 **사실과 분리 라벨링** (예: `[추정]`, `[전망]`)
- [ ] 푸터 소스 리스트가 본문에서 인용된 매체 전부를 포함

## 6. 출력 포맷 (OUTPUT)

- **산출물**: **자기완결형 HTML 1파일** (외부 의존 최소, CSS 인라인/`<style>` 내장, 이미지 없이 텍스트+CSS만; 웹폰트만 CDN 링크 허용).
  - 정적 호스팅(예: Cloudflare Pages)에 그대로 업로드 가능해야 함.
  - 반응형(모바일 1열), 인쇄 친화.
- **브랜드 적용**: §2 컬러 + §3 폰트 CSS 변수로 스타일. 헤더 바 = `--life-green`, 배지·언더라인 = 그린, 배경 화이트.
- **파일명 규칙**: must-charts 슬러그 규칙 미러링 → **`YYMMDD-lifeinve-{slug}-{hash6}.html`**
  - 예) `260701-lifeinve-substack-a1b2c3.html`, `260701-lifeinve-earnings-9f0e1d.html`
  - `YYMMDD` = 2자리연/월/일, `hash6` = 6자리 임의 해시(must-charts와 동일 관습).
- **저장 위치(로컬 드래프트)**: `C:\Users\kimfo\projects\lifeinve_charts\out\` (배포 전 드래프트).

## 부록 — 실행 순서 요약

1. 입력(`date`, `sources`, `mode`) 확정.
2. 소스 URL 병렬 페치 → 아이템 추출.
3. §1 스켈레톤에 매핑(헤더 → 오늘의 한 방 → 번호 아이템 → 보조섹션 → 푸터).
4. §4 문체(헤드=음슴체 / 본문=완전문)로 작성.
5. §5 검증 체크리스트 전수 통과.
6. §2·§3 브랜드킷으로 자기완결형 HTML 렌더.
7. §6 파일명으로 `out/` 에 저장.
8. 사용자에게 드래프트 보고 → (별도) 배포 승인 대기.
